The Strands SDK provides logging infrastructure to give visibility into its operations.

<starlight-tab-item data-label="Python"> &lt;p&gt;Strands SDK uses Python’s standard &lt;a href=&quot;https://docs.python.org/3/library/logging.html&quot;&gt;&amp;lt;code dir=&amp;quot;auto&amp;quot;&amp;gt;logging&amp;lt;/code&amp;gt;&lt;/a&gt; module. The SDK implements a straightforward logging approach:&lt;/p&gt;&lt;ol&gt; &lt;li&gt; &lt;p&gt;&lt;strong&gt;Module-level Loggers&lt;/strong&gt;: Each module creates its own logger using &lt;code dir=&quot;auto&quot;&gt;logging.getLogger(\_\_name\_\_)&lt;/code&gt;, following Python best practices for hierarchical logging.&lt;/p&gt; &lt;/li&gt; &lt;li&gt; &lt;p&gt;&lt;strong&gt;Root Logger&lt;/strong&gt;: All loggers are children of the “strands” root logger, making it easy to configure logging for the entire SDK.&lt;/p&gt; &lt;/li&gt; &lt;li&gt; &lt;p&gt;&lt;strong&gt;Default Behavior&lt;/strong&gt;: By default, the SDK doesn’t configure any handlers or log levels, allowing you to integrate it with your application’s logging configuration.&lt;/p&gt; &lt;/li&gt; &lt;/ol&gt; </starlight-tab-item><starlight-tab-item data-label="TypeScript"> &lt;p&gt;Strands SDK provides a simple logging infrastructure with a global logger that can be configured to use your preferred logging implementation.&lt;/p&gt;&lt;ol&gt; &lt;li&gt; &lt;p&gt;&lt;strong&gt;Logger Interface&lt;/strong&gt;: A simple interface (&lt;code dir=&quot;auto&quot;&gt;debug&lt;/code&gt;, &lt;code dir=&quot;auto&quot;&gt;info&lt;/code&gt;, &lt;code dir=&quot;auto&quot;&gt;warn&lt;/code&gt;, &lt;code dir=&quot;auto&quot;&gt;error&lt;/code&gt;) compatible with popular logging libraries like Pino, Winston, and the browser/Node.js console.&lt;/p&gt; &lt;/li&gt; &lt;li&gt; &lt;p&gt;&lt;strong&gt;Global Logger&lt;/strong&gt;: A single global logger instance configured via &lt;code dir=&quot;auto&quot;&gt;configureLogging()&lt;/code&gt;.&lt;/p&gt; &lt;/li&gt; &lt;li&gt; &lt;p&gt;&lt;strong&gt;Default Behavior&lt;/strong&gt;: By default, the SDK only logs warnings and errors to the console. Debug and info logs are no-ops unless you configure a custom logger.&lt;/p&gt; &lt;/li&gt; &lt;/ol&gt; </starlight-tab-item>

### Log Levels

The Strands Agents SDK uses standard log levels:

-   **DEBUG**: Detailed operational information for troubleshooting. Extensively used for tool registration, discovery, configuration, and execution flows.
    
-   **INFO**: General informational messages. Currently not used.
    
-   **WARNING**: Potential issues that don’t prevent operation, such as validation failures, specification errors, and compatibility warnings.
    
-   **ERROR**: Significant problems that prevent specific operations from completing successfully, such as execution failures and handler errors.
    
-   **CRITICAL**: Reserved for catastrophic failures.
