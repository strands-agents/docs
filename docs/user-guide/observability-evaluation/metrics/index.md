## Tool Metrics

<starlight-tab-item data-label="Python"> &lt;p&gt;For each tool used by the agent, detailed metrics are collected in the &lt;code dir=&quot;auto&quot;&gt;tool\_metrics&lt;/code&gt; dictionary. Each entry is an instance of &lt;a href=&quot;/docs/api/python/strands.telemetry.metrics/#ToolMetrics&quot;&gt;&amp;lt;code dir=&amp;quot;auto&amp;quot;&amp;gt;ToolMetrics&amp;lt;/code&amp;gt;&lt;/a&gt; that tracks the tool’s performance throughout the agent’s execution.&lt;/p&gt;&lt;p&gt;Tool metrics provide insights into:&lt;/p&gt;&lt;ul&gt; &lt;li&gt;&lt;strong&gt;Call statistics&lt;/strong&gt;: Total number of calls, successful executions, and errors&lt;/li&gt; &lt;li&gt;&lt;strong&gt;Execution time&lt;/strong&gt;: Total and average time spent executing the tool&lt;/li&gt; &lt;li&gt;&lt;strong&gt;Success rate&lt;/strong&gt;: Percentage of successful tool invocations&lt;/li&gt; &lt;li&gt;&lt;strong&gt;Tool reference&lt;/strong&gt;: Information about the specific tool being tracked&lt;/li&gt; &lt;/ul&gt;&lt;p&gt;These metrics help you identify performance bottlenecks, tools with high error rates, and opportunities for optimization. For complete details on all available properties, see the &lt;a href=&quot;/docs/api/python/strands.telemetry.metrics/#ToolMetrics&quot;&gt;&amp;lt;code dir=&amp;quot;auto&amp;quot;&amp;gt;ToolMetrics&amp;lt;/code&amp;gt; API reference&lt;/a&gt;.&lt;/p&gt; </starlight-tab-item><starlight-tab-item data-label="TypeScript"> &lt;p&gt;For each tool used by the agent, detailed metrics are collected in the &lt;code dir=&quot;auto&quot;&gt;toolMetrics&lt;/code&gt; dictionary. Each entry is a &lt;code dir=&quot;auto&quot;&gt;ToolMetricsData&lt;/code&gt; object that tracks the tool’s performance throughout the agent’s execution.&lt;/p&gt;&lt;p&gt;Tool metrics provide insights into:&lt;/p&gt;&lt;ul&gt; &lt;li&gt;&lt;strong&gt;Call statistics&lt;/strong&gt;: Total number of calls, successful executions, and errors&lt;/li&gt; &lt;li&gt;&lt;strong&gt;Execution time&lt;/strong&gt;: Total time spent executing the tool&lt;/li&gt; &lt;li&gt;&lt;strong&gt;Computed statistics&lt;/strong&gt;: The &lt;code dir=&quot;auto&quot;&gt;toolUsage&lt;/code&gt; getter adds computed &lt;code dir=&quot;auto&quot;&gt;averageTime&lt;/code&gt; and &lt;code dir=&quot;auto&quot;&gt;successRate&lt;/code&gt; fields&lt;/li&gt; &lt;/ul&gt;&lt;p&gt;These metrics help you identify performance bottlenecks, tools with high error rates, and opportunities for optimization.&lt;/p&gt; </starlight-tab-item>

## Best Practices

1.  **Monitor Token Usage**: Keep track of token usage to ensure you stay within limits and optimize costs. Set up alerts for when token usage approaches predefined thresholds to avoid unexpected costs.
    
2.  **Analyze Tool Performance**: Review tool metrics to identify tools with high error rates or long execution times. Consider refactoring tools with success rates below 95% or average execution times that exceed your latency requirements.
    
3.  **Track Cycle Efficiency**: Monitor how many iterations the agent needed and how long each took. Agents that require many cycles may benefit from improved prompting or tool design.
    
4.  **Benchmark Latency Metrics**: Monitor latency values to establish performance baselines. Compare these metrics across different agent configurations to identify optimal setups.
    
5.  **Regular Metrics Reviews**: Schedule periodic reviews of agent metrics to identify trends and opportunities for optimization. Look for gradual changes in performance that might indicate drift in tool behavior or model responses.
